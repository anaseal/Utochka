# Документация проекта: Graphic Editor for Silyanka Design

Проект представляет собой специализированный параметрический редактор для создания схем традиционных бисерных украшений (силянок). Система имитирует физику и геометрию сетчатого плетения, позволяя дизайнеру работать с логической структурой изделия, а не с простыми пикселями.

## 1. Основные сущности системы

### **Bead (Бисерина)**
Фундаментальный объект данных. Система разделяет их на две функциональные группы:
*   **NODE (Узел):** Ключевые точки пересечения нитей. Именно они определяют каркас изделия.
*   **SPAN (Соединительная):** Бисерины, заполняющие пространство между узлами на нити.

### **Grid Structure (Сетка)**
Математическая модель, построенная на чередовании рядов узлов.
*   **Четный ряд:** Базовая горизонтальная линия узловых точек.
*   **Нечетный ряд:** Ряд, имеющий горизонтальное смещение на 50% шага относительно четного ряда, что создает ромбовидную ячейку.

### **Design Map (Карта дизайна)**
Реестр состояний, хранящий информацию о цвете каждой бисерины (ID -> Color). Это позволяет сохранять узор независимо от изменения геометрии сетки.

## 2. Геометрия и логика плетения

### **Система рядов и колонн**
В отличие от таблиц, здесь расчет ведется по узловым точкам (NODE):
*   **Колонна (Column):** Вертикальная ось узлов. Параметр `width` определяет количество таких осей.
*   **Ряд (Row):** Горизонтальная ось узлов. Параметр `height` определяет количество «этажей» из ромбов. Один полный ромб по вертикали формируется за счет прохода через три уровня узлов (Верх-Центр-Низ).
*   **Самый верхний ряд (Top Edge):** Особая технологическая цепочка бисерин, соединяющая узлы самого первого ряда (r=0) строго по горизонтали. Она служит «кромкой» изделия.

### **Параметры ромба (Span Control)**
Система позволяет создавать асимметричные ячейки. Эти параметры доступны в хедере под названиями "Top Edge" и "Bottom Edge":
*   **Top Span:** Количество бисерин в верхних гранях ромба («плечи»). Регулируется в диапазоне 3–10 через кнопки ± в хедере.
*   **Bottom Span:** Количество бисерин в нижних гранях ромба («ножки»). Регулируется в диапазоне 3–10 через кнопки ± в хедере.
*   **Compression:** Вертикальное сжатие (0.25) имитирует реальную форму бисера, который под собственным весом стремится принять форму приплюснутого овала, а не идеального круга.

## 3. Функциональные возможности

### **Динамический генератор (`generator.ts`)**
При любом изменении параметров (ширина, высота, количество бисерин в пролете) система мгновенно пересчитывает координаты каждой из сотен бисерин. Алгоритм автоматически выстраивает связи между узлами, формируя целостное полотно.

### **Инструменты редактирования**
*   **Потоковое рисование:** Возможность закрашивать бисерины, просто проводя по ним курсором при зажатой кнопке мыши.
*   **Ластик (Eraser):** Второй режим инструмента — стирает цвет бисерины из Design Map, возвращая её в незакрашенное состояние. Переключение между карандашом и ластиком через кнопку ⌫ в хедере; выбор любого цвета из палитры автоматически возвращает инструмент в режим карандаша.
*   **Hitbox-адаптация:** Для удобства работы с мелкими элементами активная зона клика (11px) значительно превышает визуальный размер бисерины (6.5px).
*   **Zoom System:** Масштабирование холста от 25% до 300% с шагом 10%, с сохранением четкости векторных границ (SVG).

### **История действий (Undo/Redo)**
Система записывает историю на уровне мазков, а не отдельных кликов:
*   **Запись в историю:** Состояние Design Map фиксируется в момент начала мазка (`startDrawing`). При завершении мазка (`stopDrawing`), если карта изменилась, сохранённый снимок добавляется в стек `past`. Операция `clearAll` также добавляет снимок в историю перед очисткой.
*   **Стек истории:** Хранит до 30 снимков (`MAX_HISTORY = 30`). При переполнении самый старый снимок удаляется.
*   **Redo-стек:** Стек `future` сбрасывается при каждом новом мазке или очистке. Undo перемещает текущее состояние в `future`, Redo — возвращает обратно.
*   **Управление:** Кнопки ↩ / ↪ в хедере (с блокировкой при пустых стеках). Клавиатурные сокращения: `Ctrl+Z` — отменить, `Ctrl+Y` / `Ctrl+Shift+Z` — повторить.

### **Интерфейс управления (Header)**
Фиксированная панель инструментов в верхней части экрана:
*   **Палитра цветов:** 6 цветов для выбора (`#22d3ee`, `#e879f9`, `#ffffff`, `#ff4757`, `#2ed573`, `#eccc68`). Клик по цвету активирует режим карандаша и устанавливает этот цвет.
*   **Инструмент ластик (⌫):** Переключение между режимами карандаша и ластика. Выбор любого цвета возвращает в режим карандаша.
*   **Контролы сетки:** Группы ± кнопок для изменения Width (количество колонн), Height (количество рядов), Top Edge, Bottom Edge. Изменения пересчитываются мгновенно.
*   **Zoom контролы:** Кнопки ± для изменения масштаба (25%–300%, шаг 10%). Отображение текущего уровня масштаба в процентах.
*   **Управление историей:** Кнопки ↩ (Undo) и ↪ (Redo) для навигации по истории действий. Кнопки блокируются, если соответствующий стек пуст.
*   **Очистка (CLR):** Кнопка для полной очистки Design Map (стирает все цвета). Операция записывается в историю и может быть отменена через Undo.

### **Разметка и навигация**
*   **Интеллектуальные линейки:** CanvasRulers автоматически нумеруют ряды и колонки узловых точек. Линейка по X привязывается ко второму ряду узлов, чтобы корректно учитывать смещение ромбов.
*   **Центрирование:** Автоматический расчет ViewBox на основе максимальных координат элементов с учетом технологических отступов (offset 50px).

## 4. Аналитика и статистика (Total Count)

Система выполняет роль калькулятора материалов в реальном времени:
*   **Total Bead Count:** Общее количество всех бисерин (NODE + SPAN) в текущем проекте. Любое изменение `topSpan` или `width` мгновенно обновляет это число.
*   **Color Specification:** Модуль статистики анализирует `Design Map` и выводит список всех задействованных цветов с указанием точного количества бисерин для каждого из них. Это готовая спецификация для закупки материала.

---

## 5. Технические константы
*   **Node Radius:** 6.5px (чуть крупнее для акцента).
*   **Span Radius:** 6.0px.
*   **Spacing:** 65px (базовое расстояние между узлами).
*   **Colors:** Используются Slate-800 для фона и адаптивная палитра из 6 цветов для творчества.
